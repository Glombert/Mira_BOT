### ЭТАП 0 — Фундамент
**Статус: не начат → ДЕЛАЕМ ПЕРВЫМ**

Без этого этапа всё остальное ненадёжно.

#### 0.1 Структура папок и файлов
- [ ] Создать папки: `agents/`, `profiles/`, `tools/`, `workspace/`, `memory/`, `memory/sessions/`, `versions/`
- [ ] Создать `agents/_template.json` — базовый шаблон агента
- [ ] Создать `profiles/default.json`
- [ ] Добавить в `.gitignore`: `workspace/`, `memory/`, `versions/`, `.env`
- [ ] Обновить `requirements.txt`

#### 0.2 Защита саморедактирования
Сейчас `evolve` перезаписывает `agent.py` без проверок — один плохой патч убьёт агента.

- [ ] Бэкап перед записью → `versions/agent_YYYY-MM-DD_HH-MM-SS.py`
- [ ] Валидация через `ast.parse()` — невалидный Python не сохраняется
- [ ] Smoke-test в подпроцессе: `python agent.py --self-test`
- [ ] Авто-роллбэк: если smoke-test упал — восстановить последнюю рабочую версию
- [ ] Команда `/rollback` — ручной откат на предыдущую версию

#### 0.3 Профили пользователей (настройки интерфейса)
- [ ] Класс `Profile`: загрузка из `profiles/{name}.json`
- [ ] Запуск с профилем: `python agent.py --profile dev`
- [ ] Без аргумента — загружается `default.json`
- [ ] Профиль определяет: системный промпт, разрешённые инструменты

#### 0.4 Рефакторинг конфигурации
- [ ] Все пути и константы — в одном месте (верхушка `agent.py` или `config.py`)
- [ ] `WORKSPACE_DIR`, `BACKUP_DIR`, `MAX_HISTORY` — не разбросаны по коду

#### 0.5 Память пользователя и онбординг
- [ ] Функция `identify_user(source, **kwargs)` — определяет пользователя по интерфейсу
  - CLI: из аргумента `--user`
  - Telegram: из `user_id` сообщения
  - Web: из Telegram Login Widget
- [ ] Проверка при старте: есть файл `memory/{user_id}.json`?
- [ ] Если нет — режим онбординга:
  - Мира представляется и задаёт вопросы по одному (не анкета)
  - После разговора — API-вызов для структурирования профиля
  - Сохранение в `memory/{user_id}.json`
- [ ] Если да — загрузка профиля и приветствие по имени
- [ ] Мира сама обновляет профиль в процессе работы (узнала тариф → записала)
- [ ] Команда `/whoami` — показать что агент знает о пользователе
- [ ] Команда `/forget` — сбросить профиль, начать онбординг заново

**Результат этапа:** агент стабилен, знает кто его пользователь, помнит между сессиями.

### ЭТАП 1 — Класс Agent и Tool Calling
**Агент перестаёт советовать и начинает делать.**

#### 1.1 Рефакторинг: класс Agent
- [ ] Выделить класс `Agent` с методами: `run()`, `classify()`, `reply()`, `use_tool()`
- [ ] Конфиг агента загружается из `agents/{name}.json`
- [ ] `Agent.from_template(name)` — создать агента из шаблона
- [ ] `Agent.spawn(task_description)` — Альфа создаёт нового агента

#### 1.2 Подключение инструментов (OpenAI function calling)
- [ ] Переписать API-вызов: передавать `tools=` список доступных инструментов
- [ ] Обработчик `tool_calls` в ответе: распарсить → вызвать → вернуть результат
- [ ] Цикл: агент вызывает инструменты пока не вернёт финальный текст

#### 1.3 Базовые инструменты (`tools/file_tools.py`)
- [ ] `list_files(path)` — список файлов в workspace пользователя
- [ ] `read_file(path)` — прочитать текстовый/JSON файл
- [ ] `write_file(path, content)` — записать файл
- [ ] `delete_file(path)` — удалить (только с явным подтверждением)

Безопасность: все пути проверяются — выход за пределы `workspace/{user_id}/` запрещён.

#### 1.4 Инструмент выполнения кода (`tools/shell_tools.py`)
- [ ] `run_python(code, timeout=30)` — выполнить в подпроцессе
- [ ] Возвращает: stdout, stderr, код возврата

#### 1.5 Тест этапа
- [ ] Попросить агента создать файл → прочитать → убедиться что сработало
- [ ] Попросить посчитать что-то через `run_python`

**Результат этапа:** агент реально делает вещи, а не только рассказывает как делать.

### ЭТАП 2 — Конклав (мультиагентность)
**Альфа получает специалистов.**

#### 2.1 Оркестратор (`conclave.py`)
- [ ] Класс `Conclave`: реестр агентов, загрузка из `agents/`
- [ ] `Conclave.get(name)` — получить агента по имени
- [ ] `Conclave.list()` — список доступных агентов
- [ ] `Conclave.register(config)` — добавить нового агента в реестр

#### 2.2 Роутер (`router.py`)
- [ ] Классификатор: лёгкий вызов API определяет тип задачи
- [ ] `fast_routes` из профиля пользователя — мгновенный роутинг по ключевым словам
- [ ] Логика: `chat` → Альфа сама, `excel` → excel_specialist, `complex` → planner+executor+critic

#### 2.3 Специалисты
- [ ] `agents/excel_specialist.json` — инструменты Excel, низкая температура
- [ ] `agents/coder.json` — `run_python`, написание и проверка кода
- [ ] `agents/planner.json` — декомпозиция задач, не выполняет сам
- [ ] `agents/critic.json` — проверяет результат, возвращает ok или список проблем

#### 2.4 Саморасширение
- [ ] Альфа может сгенерировать новый `agents/{name}.json` под новую задачу
- [ ] Новый агент наследует ограничения профиля пользователя
- [ ] Факт создания логируется в `memory/decisions.log`
- [ ] Защита: Альфа не может создать агента с ролью `alpha`

**Результат этапа:** сложные задачи разбиваются на специалистов прозрачно для пользователя.

### ЭТАП 3 — Excel и сценарий мамы
**Главная пользовательская ценность.**

#### 3.1 Excel инструменты (`tools/excel_tools.py`)
- [ ] `excel_read(path, sheet=None)` — данные листа в JSON
- [ ] `excel_write(path, data, sheet=None)` — записать данные
- [ ] `excel_fill(template_path, data, output_path)` — заполнить шаблон, сохранить форматирование

Зависимости: `openpyxl`, `xlrd`

#### 3.2 Сценарий "показания счётчиков"
Конкретный рабочий флоу:

1. Мама отправляет файл боту в Telegram (или загружает в Web UI)
2. Файл сохраняется в `workspace/tg_{user_id}/input/`
3. Агент читает шаблон из `workspace/tg_{user_id}/templates/`
4. Считает разницу (текущие − предыдущие) × тариф из профиля мамы
5. Заполняет счета, сохраняет в `workspace/tg_{user_id}/output/`
6. Отправляет готовый файл обратно в Telegram / предлагает скачать

- [ ] Согласовать с мамой структуру входного файла (нужен реальный пример)
- [ ] Получить шаблон счёта (реальный файл)
- [ ] Реализовать и протестировать на реальных данных
- [ ] Обработка ошибок: пропущенные данные, нулевые показания, новые квартиры

**Результат этапа:** мама может работать без помощи.

- [ ] `GET /download/{filename}` — скачать из output

- [ ] Адаптивный дизайн (работает с телефона)

#### 4.3 Запуск и деплой
- [ ] `uvicorn web.app:app --host 0.0.0.0 --port 8000`
- [ ] Systemd-юниты для бота и веб-сервера

**Результат этапа:** мама пишет боту в Telegram или открывает браузер — работает.

## Порядок работы (как двигаемся)

1. Берём следующий незакрытый пункт
2. Я пишу код, ты запускаешь
3. Говоришь что работает, что нет — итерируем
4. Ставим галочку, идём дальше

**При каждом новом сеансе** — закинь актуальный `agent.py` и этот `PLAN.md`.
