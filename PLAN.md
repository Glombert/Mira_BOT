## Принятые архитектурные решения

### Самоизменение и пользователи
- `agent.py` — единое ядро для всех пользователей. Мира не меняет ядро под конкретного пользователя.
- `/evolve` — инструмент только для профиля `dev`. В остальных профилях заблокирован.
- Мира меняет: конфиги агентов в `agents/`, память пользователя в `memory/`.
- Новые агенты в Конклав (`agents/новый.json`) создаются Альфой — это не изменение ядра.

### Облачное хранение памяти (rclone + Google Drive)
- Используем **rclone** для синхронизации с Google Drive.
- Синхронизируется: `memory/` (профили, сессии), опционально `workspace/`.
- Команда `/cloud sync` — ручная синхронизация.
- Команда `/cloud restore` — восстановление на новом сервере.
- Настраивается в `.env`: `RCLONE_REMOTE=gdrive:mira_memory`
- Реализация в **Этапе 5** (долгая память).

### Структура workspace пользователя
```
workspace/
└── {user_id}/
    ├── inbox/      ← пользователь кидает файлы сюда
    ├── output/     ← Мира кладёт результаты сюда
    └── temp/       ← временные файлы, автоочистка через 7 дней
```
- Мира работает только внутри `workspace/{user_id}/` — выход за пределы запрещён.
- Автоочистка `temp/` запускается при каждом старте агента.

### Порядок реализации Tool Calling (изменено в v2.4)
Мы идём снизу вверх: сначала инструменты, потом их подключение к API,
потом рефакторинг в класс. Причина: рефакторинг без рабочих инструментов —
абстрактная работа, легко ошибиться. Рабочие инструменты с тестами —
надёжная база для класса Agent.

```
1.3 → file_tools.py  (сделано ✓)
1.4 → shell_tools.py (сделано ✓)
1.2 → function calling в API
1.1 → класс Agent (рефакторинг последним)
```

```

│
```

### ЭТАП 1 — Инструменты и Tool Calling
**Статус: почти готов — остался рефакторинг в класс Agent**

#### 1.3 + 1.4 Инструменты ✓

- [x] `tools/file_tools.py` — list_files, read_file, write_file
  - sandbox: Мира не выходит за пределы workspace/{user_id}/
  - ограничение на размер файла: 1 MB
  - overwrite=False по умолчанию (защита от случайной перезаписи)
- [x] `tools/shell_tools.py` — run_python
  - подпроцесс с таймаутом 30 сек
  - вывод обрезается до 8000 символов
  - рабочая директория: workspace/{user_id}/

#### 1.2 Подключение инструментов к API (function calling) ✓

- [x] TOOL_SCHEMAS — JSON-схемы всех четырёх инструментов
- [x] execute_tool() — диспетчер: принимает имя и аргументы, возвращает JSON-строку
- [x] Главный цикл: tool calling loop с MAX_TOOL_ROUNDS=5
- [x] Передача `tools=` и `tool_choice="auto"` в каждый запрос
- [x] Обработка tool_calls: выполнение + возврат результата с role="tool"

#### 1.1 Рефакторинг: класс Agent ← СЛЕДУЮЩИЙ

- [ ] Выделить класс `Agent` с методами: `run()`, `classify()`, `reply()`, `use_tool()`
- [ ] Конфиг агента загружается из `agents/{name}.json`
- [ ] Безопасность: `/evolve` доступен только в профиле `dev` (через `profile.can_use()`)
- [ ] Git-функции вынести в `tools/git_tools.py` (уберём из agent.py)

Почему последним: теперь мы точно знаем как работает цикл,
и рефакторинг будет осознанным, а не угадыванием.
